## 📋 Overview

This system detects intoxicated drivers in real-time by combining **computer vision** (facial analysis via MobileNetV3) with **IoT sensor data** (MQ3 alcohol sensor). When drunk driving is detected continuously, the system:

1. 🔊 Activates physical alarms (speaker + warning light)
2. 📱 Sends Telegram notifications with photo evidence
3. 📝 Logs violations with timestamps for fleet management
4. 📊 Displays violation history on a web dashboard

## 🏗️ Architecture

```
Camera Frame → Face Extraction (MediaPipe) → TFLite Inference (MobileNetV3)
                                                        ↓
MQ3 Sensor → Arduino (Serial) ──────────→ Decision Fusion (OR logic)
                                                        ↓
                                              Alert System
                                           ├── 🔊 Speaker
                                           ├── 💡 Warning Light  
                                           ├── 📱 Telegram Alert
                                           └── 📝 CSV Logger
```

## 📊 Results

| Metric | Value |
|--------|-------|
| **Model** | MobileNetV3Small (transfer learning) |
| **Training** | 2-phase (frozen → fine-tune) |
| **Model Size** | ~3 MB (TFLite quantized) |
| **Inference** | <100ms on Raspberry Pi 5 |
| **Input** | 224×224 RGB face images |
| **Evaluation** | Accuracy, F1, ROC-AUC, Precision-Recall, FNR |

> Full evaluation metrics, Grad-CAM visualizations, and threshold analysis are generated by `scripts/evaluate.py`.

## ✨ Key Features

| Feature | Description |
|---------|-------------|
| **Two-Phase Training** | Freeze backbone → fine-tune top layers |
| **Validation Fix** | No augmentation on validation/test sets |
| **Comprehensive Evaluation** | ROC-AUC, PR curve, threshold analysis, Grad-CAM |
| **Knowledge Distillation** | Compress teacher → tiny student (17x smaller) |

| **Hyperparameter Tuning** | Optuna TPE Bayesian optimization |

| **Structured Logging** | Python `logging` module throughout |

## 🚀 Quick Start

### 1. Clone & Install

```bash
git clone https://github.com/your-username/Drunk_Detection_System.git
cd Drunk_Detection_System

# Create virtual environment
python -m venv venv
source venv/bin/activate  # Linux/Mac
# venv\Scripts\activate   # Windows

# Install package
pip install -e ".[dev]"
```

### 2. Configure Environment

```bash
cp .env.example .env
# Edit .env with your Telegram token and other settings
```

### 3. Train the Model

```bash
python scripts/train.py \
    --train-dir ./data/train \
    --val-dir ./data/val \
    --test-dir ./data/test \
    --mlflow
```

### 4. Evaluate

```bash
python scripts/evaluate.py \
    --model outputs/models/drunk_detection_model.keras \
    --test-dir ./data/test \
    --grad-cam
```

### 5. Export for Edge Deployment

```bash
python scripts/export_tflite.py \
    --model outputs/models/drunk_detection_model.keras \
    --optimize \
    --verify
```

### 6. Deploy on Raspberry Pi

```bash
# Copy model to Raspberry Pi
scp outputs/models/drunk_detection.tflite pi@raspberrypi:~/models/

# On Raspberry Pi:
cd deployment/raspi

# Run health check first
python main.py --health-check

# Run benchmark
python main.py --benchmark

# Start detection
python main.py
```

### 7. Advanced ML (Phase 6)

```bash
# Knowledge Distillation — compress model 17x
python scripts/distill.py \
    --teacher outputs/models/drunk_detection_model.keras \
    --train-dir ./data/train --val-dir ./data/val \
    --student-type tiny --temperature 5.0

# Quantization-Aware Training — better INT8 accuracy
python scripts/qat_export.py \
    --model outputs/models/drunk_detection_model.keras \
    --train-dir ./data/train --val-dir ./data/val --full-int8

# Hyperparameter Tuning — find optimal config
python scripts/tune.py \
    --train-dir ./data/train --val-dir ./data/val --n-trials 30
```

### 8. Launch Dashboard

```bash
cd deployment/dashboard
python app.py
# Open http://localhost:5000
```

## 🎯 Technical Highlights (Interview Talking Points)

1. **Why MobileNetV3Small?** — Designed for mobile/edge: inverted residuals, squeeze-excitation, h-swish activation. ~2.5M params vs ResNet50's 25M.

2. **Why 2-phase training?** — Phase 1 trains the head (prevents catastrophic forgetting). Phase 2 fine-tunes top layers with 10x lower LR for domain adaptation.

3. **Validation augmentation bug (FIXED)** — Original code applied augmentation to validation data, biasing evaluation metrics. Now only training uses augmentation.

4. **Safety-critical threshold** — FNR (False Negative Rate) is prioritized over accuracy. A drunk driver classified as sober is dangerous. Threshold analysis finds the optimal operating point.

5. **Deployment pipeline** — Keras → TFLite (quantized) → Raspberry Pi. Model size reduced from ~10MB to ~3MB with minimal accuracy loss.

6. **What would I improve?** — Online learning for distribution shift, face recognition for driver ID, model drift detection, A/B testing for thresholds.
